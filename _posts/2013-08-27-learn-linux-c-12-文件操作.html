Linux文件结构

    @1：一些都是文件
    @2：open read write close ioctl
    @3：三个特殊文件 /dev/console /dev/rrt /dev/null

系统调用和设备驱动程序

    @open：打开文件或设备
    @read：行打开的文件或设备里读取数据
    @write：向文件或设备里卖弄写数据
    @close：关闭设备或文件
    @ioctl：把控制信息传递给设备驱动程序

库函数

    @系统调用的效率很低，因为要切换到内核代码。改善：少用or一次多用
    @系统调用的效率很低，因为硬件做了限制

    @open系统调用原型
    

    

#include <fcntl.h>
#include <sys/types.h>
#include <sys/stat.h>
int open(const char* path, int oflags);
int open(const char* path, int oflags, mode_t mode);

        oflags: |实现
          O_RDONLY   只读
          O_WRONLY   只写
          O_RDWR     读写
          O_APPEND   追加写入
          O_TRUNC    把文件清空再写入
          O_CREAT    如果需要，按照mode中给访问模式创建文件
          O_EXCL     与O_CREAT一起使用，确保调用者创建文件
        mode标识：|实现
          S_IRUSR：读权限，  文件属主
          S_IWUSR：写权限，  文件属主
          S_IXUSR：执行权限，文件属主
          S_IRGSR：读权限，  文件属组
          S_IWGSR：写权限，  文件属组
          S_IXGSR：执行权限，文件属组
          S_IROSR：读权限，  其他用户
          S_IWOSR：写权限，  其他用户
          S_IXOSR：执行权限，其他用户
        另：creat相当于oflags=(O_CWDONLY|O_CREAT|O_TRUNC)
        umask:user mask权限设置的东西
